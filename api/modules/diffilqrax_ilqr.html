diffilqrax.ilqr.ilqr_solver(model, params, Us_init, max_iter=40, convergence_thresh=1e-06, alpha_init=1.0, verbose=False, use_linesearch=True, **linesearch_kwargs)[source]#

Solve the iterative Linear Quadratic Regulator (iLQR) problem.

The non-linear system is approximated to be a local LQR problem. The optimal control gains are iteratively added to the x and u perturbations until the cost converges to a local optimum. Convergence speed can be improved by using line search to find the optimal step size.

Parameters:

  • model (System) – The system model.

  • params (iLQRParams) – The parameters of the system.

  • Us_init (Array) – The initial control trajectory.

  • max_iter (int, optional) – The maximum number of iterations, by default 40.

  • convergence_thresh (float, optional) – The convergence threshold, by default 1e-6.

  • alpha_init (float, optional) – The initial step size for the forward pass, by default 1.0.

  • verbose (bool, optional) – Whether to print debug information, by default False.

  • use_linesearch (bool, optional) – Whether to use line search for the forward pass, by default True.

  • **linesearch_kwargs – Additional keyword arguments for the line search.

Returns:

A tuple containing the final state trajectory, control trajectory, and the adjoint variables. Also returns the total cost of the trajectory and the cost history.

Return type:

Tuple[Tuple[Array, Array, Array], float, Array]

diffilqrax.ilqr.linesearch(update, Ks, Xs_init, Us_init, alpha_init, cost_init, expected_dJ, beta=0.5, max_iter_linesearch=12, tol=0.1, alpha_min=1e-06, verbose=False)[source]#

Implementation of the line search backtracking algorithm for iLQR algorithm.

Each iteration of the line search algorithm performs a forward pass with the new control inputs defined by the gains and an alpha value. The change in cost is compared to the expected change in cost and the alpha is selected based on the ratio of the two values above a given tolerance. Otherwise, the alpha value is reduced by a factor of beta.

References

Tassa, Y., Erez, T., & Todorov, E. (2012). Synthesis and stabilization of complex behaviors through online trajectory optimization. IEEE. https://doi.org/10.1109/iros.2012.6386025

Parameters:

  • update (Callable) – Rollout function which returns new Xs, Us and cost.

  • Ks (Gains) – Gains obtained from the LQR controller.

  • Xs_init (Array) – State trajectory.

  • Us_init (Array) – Input trajectory.

  • alpha_init (float) – Initial alpha value.

  • cost_init (float) – Cost of initial trajectory.

  • expected_dJ (CostToGo) – Expected change in cost from LQR controller.

  • beta (float, optional) – Reduction factor for alpha, by default 0.5.

  • max_iter_linesearch (int, optional) – Maximum iterations of linesearch, by default 12.

  • tol (float, optional) – Tolerance of ratio of actual to expected cost change to accept alpha value, by default 0.1.

  • alpha_min (float, optional) – Minimum alpha value, by default 1e-6.

  • verbose (bool, optional) – Whether to print debug information, by default False.

Returns:

Returns the updated state and control trajectories, the alpha value, the total cost, and the cost history.

Return type:

Tuple[Tuple[Array, Array], float, float, Array]
